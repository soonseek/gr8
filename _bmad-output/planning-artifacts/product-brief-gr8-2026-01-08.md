# Product Brief: gr8

---

## Executive Summary

**gr8**은 자동매매 시장의 투명성 혁명을 가져오는 탈중앙화 플랫폼입니다.

기존 자동매매 서비스들은 불투명한 백테스트 결과, 검증 불가능한 성과 주장, 그리고 사용자의 전략을 플랫폼이 소유하는 폐쇄적 구조를 가지고 있습니다. 사용자들은 "이 백테스트가 진짜야?"라는 불신과 "내 전략을 왜 너희가 가져가?"라는 종속감 속에서 서비스를 이용해야 합니다.

gr8은 **100% 오픈소스**와 **블록체인 기반 검증**으로 이 문제를 해결합니다. 모든 소스가 공개되어 있어 누구나 LLM이나 개발자에게 작동 원리를 검증받을 수 있으며, 전략의 실행 결과는 변조 불가능한 블록체인에 기록됩니다.

특히, gr8은 **지식 공유 생태계**를 통해 사용자가 자신의 전략을 공유할 때 3계층 보상 시스템(직접 보상, 파생 보상, 거버넌스)으로 보상받습니다. 이는 "지식은 공유할수록 가치가 커진다"는 철학을 실현합니다.

초기에는 본인과 유튜브 구독자들이 입증 가능한 전략을 만들어가며 커뮤니티를 형성하고, 장기적으로는 LLM 코파일럿이 워크플로우를 자동 생성하고 검증하는 혁신적인 단계까지 나아갈 것입니다.

## Core Vision

### Problem Statement

자동매매 시장은 **세 가지 핵심 문제**로 고통받고 있습니다:

1. **투명성 부재**: 기존 서비스들은 백테스트와 성과를 검증할 수 없는 블랙박스 형태로 제공합니다. 사용자들은 "이게 진짜야?"라고 의심할 수밖에 없습니다.

2. **플랫폼 종속**: 사용자가 만든 전략을 플랫폼이 소유하며, 사용자는 언제든 서비스에서 퇴출당할 수 있습니다. Web2 중앙화 서비스의 구조적 한계입니다.

3. **높은 진입장벽**: TradingView, ProRealTime, 3 Commas 같은 기존 플랫폼들은 코딩 지식이 필요하거나 직관적이지 않아 일반 투자자들이 진입하기 어렵습니다.

### Problem Impact

이 문제들은 다음과 같은 영향을 미칩니다:

- **사용자의 불신**: 자동매매 시장 전체에 대한 신뢰도가 낮습니다.
- **지식의 폐쇄성**: 성공적인 전략을 가진 소수(공급자)와 이를 필요로 하는 다수(수요자) 사이의 정보 격차가 심화됩니다.
- **시장의 성장 저해**: 잠재적 사용자들이 진입장벽에 막혀 시장이 확장되지 못합니다.

### Why Existing Solutions Fall Short

**기존 자동매매 서비스들 (3 Commas, 중소규모 업자들)의 한계:**

1. **지식 공유 기능 부재**: 사용자가 자신의 전략을 공유하고, 타인의 전략을 개선하며, 그 과정에서 보상받는 생태계가 없습니다.

2. **진입장벽이 높음**: 직관적이지 않은 인터페이스와 복잡한 설정 과정으로 인해 일반 투자자들이 사용하기 어렵습니다.

3. **투명성 부족**: 오픈소스가 아니며, 백테스트 결과와 실행 성과를 검증할 방법이 없습니다.

**노코드 자동화 툴 (n8n, Make)의 한계:**
- 트레이딩 도메인에 특화되지 않아 "개발자를 위한 장난감"처럼 보입니다.
- 백테스팅, 리스크 관리, 실전 실행 등 자동매매에 필요한 기능이 부족합니다.

### Proposed Solution

gr8은 **투명하게 검증 가능하고, 공유하면 보상받는 자동매매 플랫폼**입니다.

**핵심 기능:**

1. **100% 오픈소스**: 모든 소스 코드가 공개되어 있어 누구나 작동 원리를 검증할 수 있습니다. 개발자가 아니더라도 LLM에게 물어보면 모든 것을 설명받을 수 있습니다.

2. **블록체인 기반 검증**: 전략의 백테스트 결과와 실행 성과는 변조 불가능한 블록체인에 기록됩니다. 기존 자동매매 서비스들과 완전히 다른 톤과 신뢰를 제공합니다.

3. **지식 공유 생태계**: 사용자가 자신의 전략을 공유할 때 3계층 보상 시스템(직접 보상, 파생 보상, 거버넌스)으로 보상받습니다. "지식은 공유할수록 가치가 커진다"는 철학을 실현합니다.

4. **직관적 노코드 인터페이스**: n8n 스타일의 노드-엣지 에디터로 코딩 없이 복잡한 전략을 구성할 수 있습니다.

5. **미래 혁신 (차후 페이즈)**: LLM 코파일럿이 워크플로우를 자동으로 생성하고 검증하는 단계까지 나아갈 것입니다.

### Key Differentiators

| 구분 | 기존 솔루션 (3 Commas 등) | gr8 |
|------|------------------------|-----|
| **투명성** | 블랙박스, 검증 불가 | 100% 오픈소스, LLM으로 검증 가능 |
| **검증** | 센트럴리즘 (서버에 의존) | 블록체인 기반 변조 불가능 기록 |
| **지식 공유** | ❌ 없음 | ✅ 3계층 보상 시스템 |
| **진입장벽** | 높음 (코딩/복잡한 UI) | 낮음 (직관적 노코드) |
| **소유권** | 플랫폼이 소유 | 사용자가 완전 소유 |
| **미래 혁신** | 제한적 | LLM 코파일럿 자동 생성/검증 |

**gr8만의 무형 자산:**
- **빌드 인 퍼블릭**: 유튜브를 통해 개발 과정을 투명하게 공유하며 초기 커뮤니티 형성
- **철학적 명확성**: "손매매는 도박, 자동매매 전략 공유는 가치 증대"라는 명확한 철학
- **타이밍**: 노코드 자동화($35.61B), Web3($6.11B), 시스템 트레이딩의 3중 교차점에 위치

- **역할**: "이 전략 도박이야!"라고 신고하는 사람
- **동기**: 건전한 생태계 유지
- **보상**: 신고가 정당하면 보상

### User Journey

#### 김준혁의 여정 (탈출을 원하는 38세 트레이더)

**1. 발견 (Discovery)**
- 유튜브에서 "gr8 개발 라이브" 우연히 시청

- Discord에 가입

**2. 온보딩 (Onboarding)**
- 가입: 지갑 연결 (Web3 친숙하지 않아도 가이드 잘 되어 있음)
- 튜토리얼: "드래그 앤 드롭이네? 쉽네"
- 템플릿 탐색: "이거 6개월 동안 30% 수익? 대박"
- 첫 번째 전략 복제: "파라미터만 조정하면 돼?"

**3. 핵심 사용 (Core Usage)**
- 첫 주: 템플릿 가져와서 백테스트 10번 돌려봄
- 두 번째 주: 자신만의 전략 직접 만듦 (RSI + MACD 조합)
- "내가 코딩 없이 전략을 만들었다! 이거 대박이네"
- 첫 실전: 작은 금액으로 1주일 실행 → +2% 수익
- **"이거 진짜 되네!" (Aha! Moment)**

**4. 성공의 순간 (Success Moment)**
- 1달 후: 실전 수익 +8%, Drawdown -3%
- "이제 내가 안 보고도 돈을 벌어"
- 와이프: "당신 요즘 차트 안 보네?"
- "그래, 이제 내 삶을 되찾았어"
- 주말에 가족 소풍 감 🏞️

**5. 장기 사용 (Long-term)**
- 3달 후: 자신의 전략을 공개
- 첫 번째 수익화: 10명이 복제, 50,000원 수익
- "내 지식이 돈이 되네!"
- gr8 커뮤니티에서 활동적인 멤버로 성장
- **"이제 나도 도박꾼이 아니야, 시스템 트레이더야"**

## Success Metrics

### 사용자 성공 순간 (User Success Moments)

gr8 사용자가 경험하는 **4가지 핵심 성공 모멘트**:

#### 1. 첫 번째 백테스트 성공 🎯
- **행동**: 노드-엣지 에디터로 첫 전략 생성 → 백테스트 실행
- **결과**: "+15% 수익! MDD -5%!" 확인
- **감정**: "와, 내가 코딩 없이 이걸 만들었다?"
- **지표**: 첫 주 차에 90% 이상의 사용자가 최소 1회 백테스트 완료

#### 2. 첫 실전 수익 💰
- **행동**: 작은 금액으로 첫 실전 실행 (예: 10만 원)
- **결과**: 1주일 뒤 +3% 수익 확인
- **감정**: "이거 진짜 되네! 손매매와는 달라"
- **지표**: 첫 달 차에 60% 이상의 사용자가 실전 실행, 평균 +2% 이상 수익

#### 3. 첫 번째 수익화 💎
- **행동**: 자신의 전략을 공개 (마켓플레이스에 퍼블리시)
- **결과**: 10명이 복제해서 50,000원 수익 획득
- **감정**: "내 지식이 돈이 되네!"
- **지표**: 3달 차에 20% 이상의 사용자가 자신의 전략 공개

#### 4. 생활 회복 🏞️
- **행동**: 더 이상 실시간 차트를 보지 않음, 주말에 가족과 소풍
- **결과**: 도파민 중독에서 탈출, 삶의 질 개선
- **감정**: "이제 내 삶을 되찾았어"
- **지표**: 설문조사에서 80% 이상이 "삶의 질이 개선되었다"고 응답

#### 1. 참여도 (Engagement) 📊

- **주간 백테스트 실행 수**: 활성 사용자 1인당 주간 백테스트 수

**측정 방법:**
- 사용자 세션 로그 분석
- 전략 생성/백테스트 이벤트 추적
- 주간 보고서 자동 생성

---

#### 2. 생태계 활성화 (Ecosystem Activation) 🌱

**핵심 지표:**
- **공유된 전략 수**: 마켓플레이스에 공개된 총 전략 수

- **포크 대 원본 비율**: 포크된 전략 수 / 공유된 전략 수
  - 목표: 3개월 0.25 → 12개월 0.5

- **크리에이터 수**: 적어도 1개 이상 전략을 공유한 사용자 수
  - 목표: 3개월 30명 → 12개월 200명

**측정 방법:**
- 마켓플레이스 데이터베이스 쿼리
- 포크 이벤트 추적
- 크리에이터 활동 대시보드

#### 3. 수익 (Revenue) 💰

**측정 방법:**

## 입증 기준 (Validation Criteria)

**"수익이 입증되면 누구나 쓴다"** - 입증을 위한 구체적 기준:

### 3개월 입증 기간
- **기간**: 런치 후 정확히 3개월
- **마감일**: Day 90

### 최소 100명의 수익 사용자
- **정의**: 최소 1회 이상 실전 실행에서 **수익**을 본 사용자
- **기준**: 거래 수수료, 슬리피지를 제고하고 순수익 +인 경우
- **검증**: 블록체인에 기록된 실행 결과로 증명

### 월 5% 이상의 수익
- **수익률 계산**: (최종 자산 - 초기 자산) / 초기 자산 × 100
- **기간**: 월간 복리 (Compound Monthly Growth Rate)
- **목표**: 최소 100명 중 80명(80%)이 월 5% 이상 수익
- **변동성**: MDD -10% 이내 (리스크 관리 준수)

### 입증 성공 시나리오
**Day 90:**
- 100명의 사용자가 실전 실행
- 85명이 수익 (85% 승률)
- 평균 수익률: 월 7.2%
- MDD: 평균 -6.8%

→ **"입증 완료! 이제 대규모 마케팅 시작 가능!"**

## MVP Scope

### Core Features (MVP 핵심 기능)

#### 1. 노코드 워크플로우 에디터 🎨
**목적:** 사용자가 코딩 없이 복잡한 트레이딩 전략을 구성할 수 있도록

**핵심 기능:**
- **n8n 스타일 노드-엣지 인터페이스**
  - 직관적인 드래그 앤 드롭
  - 시각적 전략 구조 표현
  
- **기본 트레이딩 빌딩 블록**
  - **트리거 노드**: 시간, 가격 변동 시작 조건
  - **데이터 소스 노드**: 시장 데이터 조회 (OHLCV)
  - **조건 노드**: IF/AND/OR 논리 연산
  - **액션 노드**: 매수/매도, 포지션 관리

- **전략 저장/로드**
  - 로컬 스토리지
  - 온체인 저장 (Web3 통합)

---

#### 2. 백테스팅 엔진 ⚙️
**목적:** 사용자가 자신의 전략을 과거 데이터로 검증할 수 있도록

**핵심 기능:**
- **히스토리컬 데이터 접근**
  - Binance OHLCV 데이터 (1분, 5분, 1시간, 일간 타임프레임)
  - 최소 1년 치 데이터

- **백테스트 실행**
  - 캔들 순회 방식 시뮬레이션
  - 사용자 정의 기간 설정
  - 병렬 실행 지원 (여러 전략 동시 백테스트)

- **성과 지표 자동 계산**
  - 총 수익률 (%)
  - CAGR (연평균 성장률)
  - MDD (최대 낙폭)
  - 승률 (승/패 비율)
  - Sharpe Ratio (리스크 조정 수익률)

---

#### 3. 고급 차트 시각화 📊
**목적:** 백테스팅 결과를 시뮬레이션처럼 생동감 있게 표현하여 신뢰도 제고

**핵심 기능:**
- **실시간 차트 재생**
  - 백테스팅 결과를 캔들 단위로 재생
  - "플레이" 버튼으로 전체 과정 시뮬레이션
  - 속도 조절 (1x, 2x, 5x, 10x)

- **액션 히스토리 마크**
  - 차트상에 매수/매도 포지션 명확히 표시
  - Buy: 초록색 화살표 ↑
  - Sell: 빨간색 화살표 ↓
  - 마우스 오버 시 상세 정보 툴팁

- **성과 대시보드**
  - 수익 곡선 (PnL Curve)
  - Drawdown 그래프
  - 누적 수익률 차트
  - 거래 내역 테이블

**왜 필수인가?**
- "이 백테스트 진짜야?"라는 불신 해소
- 시뮬레이션처럼 돌아가야 신뢰도 있음
- 사용자가 과정을 눈으로 확인하고 학습

---

#### 4. 기본 Web3 통합 🔗
**목적:** 전략을 탈중앙화 방식으로 저장/공유하고 투명성 확보

**핵심 기능:**
- **지갑 연결**
  - Metamask, WalletConnect 지원
  - 간단한 연결 가이드 (Web3 비친숙자도)

- **전략 온체인 저장**
  - 사용자가 생성한 전략을 블록체인에 기록
  - IPFS 해시로 전략 구조 저장
  - 변조 불가능한 소유권 증명

- **기본 토큰 인센티브 (직접 보상만)**
  - 전략 공유 시 고정 토큰 보상
  - 복제 시 토큰 결제
  - (파생 보상, 거버넌스는 Phase 2)

---

#### 5. 템플릿 마켓플레이스 (기본) 🏪
**목적:** 사용자가 다른 사람의 전략을 발견하고 복제할 수 있도록

**핵심 기능:**
- **공개 전략 목록 조회**
  - 전략 카드 형태로 표시
  - 필터링 (수익률, MDD, 인기도)
  - 검색 기능

- **전략 상세 보기**
  - 전략 설명
  - 백테스팅 결과 요약
  - 전략 구조 (노드-엣지) 미리보기

- **원클릭 복제**
  - "복제하기" 버튼 클릭
  - 토큰 결제 (직접 보상만)
  - 내 전략 목록에 자동 추가

- **전략 공개 (퍼블리시)**
  - 사용자가 자신의 전략을 공개
  - 가격 설정 (무료 또는 유료)
  - 공개 시 토큰 보상 받음

### MVP 개발 철학

**"학습 중심의 MVP (Learning-Focused MVP)"**

gr8의 MVP는 완벽한 제품이 아니라, **사용자 니즈를 배우기 위한 도구**입니다.

**핵심 가정 검증:**
1. 사용자들은 노코드 인터페이스로 전략을 만들 수 있는가?
2. 백테스팅 시뮬레이션으로 신뢰도를 얻을 수 있는가?
3. Web3 투명성이 실제로 중요한가?
4. 사람들은 자신의 전략을 공유할 의향이 있는가?

**MVP에서 배울 것:**
- 어떤 노드 조합이 가장 인기 있는가?
- 백테스팅 후 실전 실행 전환율은 얼마인가?
- 사용자들은 얼마나 자주 전략을 수정하는가?
- 어떤 가격 포인트에서 전략 복제가 일어나는가?

**성공하면:**
- Phase 2 지식 공유 생태계로 진행
- 실전 실행 기능 추가
- 파생 보상, 거버넌스 도입

**실패하면:**
- 사용자 니즈 재분석
- 피벗 또는 방향 조정
- "백테스팅만으로는 부족하다"는 인사이트 얻기
